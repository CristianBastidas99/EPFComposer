Task: Definir los requerimientos de sistema para los casos de uso de negocio
Documentar los requisitos específicos del sistema que se derivan de los casos de uso de negocio identificados.
Purpose
Especificar los requisitos funcionales y no funcionales del sistema relacionados con los casos de uso de negocio.
Relationships
RolesPrimary Performer:
  • Analista de Requerimientos
Additional Performers:
InputsMandatory:
  • Sesiones de trabajo con stakeholders y análisis de requisitos
    		Optional:
    • None
    Outputs
    • Diagrama de casos de uso de negocio
      Main Description
      En esta tarea, se definirán y documentarán los requerimientos de sistema que son necesarios para dar soporte a los casos de uso de negocio. Se especificarán los comportamientos esperados, las interfaces de usuario, las reglas de negocio y cualquier otro requisito funcional o no funcional relacionado. Al definir los requerimientos de sistema para los casos de uso de negocio, se establecen los criterios claros y medibles para el desarrollo y evaluación del sistema.
      Steps
      Revisar el diagrama de casos de uso de negocio para comprender los casos de uso identificados.
      Identificar los requerimientos funcionales y no funcionales para cada caso de uso de negocio, como la entrada y salida de datos, la validación de datos, la seguridad, la escalabilidad y el rendimiento.
      Documentar los requerimientos del sistema para cada caso de uso de negocio y validarlos con los interesados.
      Priorizar los requerimientos del sistema y establecer un plan para su implementación.
