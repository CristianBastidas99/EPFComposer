Task: Diseñar casos de prueba
Crear una serie de casos de prueba que cubran diferentes escenarios y condiciones para probar el sistema.
Purpose
Crear casos de prueba que permitan evaluar y verificar el funcionamiento del sistema.
Relationships
RolesPrimary Performer:
  • Especialista en Pruebas
Additional Performers:
InputsMandatory:
  • Requisitos funcionales y no funcionales, especificaciones de casos de prueba, entorno de pruebas y recursos disponibles
    		Optional:
    • None
    Outputs
    • Plan de Pruebas
      Main Description
      En esta tarea, se diseñarán casos de prueba que abarquen diferentes funcionalidades y características del sistema. Estos casos de prueba deben ser diseñados de manera exhaustiva para garantizar una cobertura adecuada de los requisitos y escenarios relevantes. Se definirán los pasos de prueba, los datos de entrada y los resultados esperados para cada caso de prueba. El diseño de casos de prueba es fundamental para asegurar que el sistema funcione correctamente y cumpla con los requisitos establecidos.
      Steps
      Revisar el Plan de Pruebas y los escenarios definidos por el Desarrollador BackEnd.
      Identificar las diferentes combinaciones de entradas y condiciones necesarias para probar cada escenario.
      Diseñar casos de prueba que cubran todas las combinaciones y condiciones identificadas.
      Documentar los casos de prueba en el Plan de Pruebas, incluyendo la descripción, los pasos y las expectativas de resultados esperados.
