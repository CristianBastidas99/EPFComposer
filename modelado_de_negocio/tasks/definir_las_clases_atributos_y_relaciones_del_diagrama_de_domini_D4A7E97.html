Task: Definir las clases, atributos y relaciones del diagrama de dominio
Definir las clases, atributos y relaciones que conforman el diagrama de dominio del proyecto.
Purpose
Establecer la estructura y las conexiones entre las entidades clave del dominio en un diagrama visual.
Relationships
RolesPrimary Performer:
  • Analista de Negocio
Additional Performers:
InputsMandatory:
  • Entrevistas con expertos en el dominio y análisis de documentación
    		Optional:
    • None
    Outputs
    • Diagrama de Dominio
      Main Description
      En esta tarea, se llevará a cabo la definición de las clases, sus atributos y las relaciones que existen entre ellas en el diagrama de dominio. Se identificarán las entidades principales del dominio del proyecto y se establecerán sus características y conexiones relevantes. Al definir las clases, atributos y relaciones del diagrama de dominio, se crea una representación visual que ayuda a comprender la estructura y la lógica subyacente del sistema o proyecto.
      Steps
      Revisar la documentación de negocio y entender los conceptos involucrados.
      Identificar las clases que son necesarias para modelar el dominio.
      Definir los atributos y métodos de cada clase y establecer las relaciones entre ellas.
      Verificar que el diagrama sea coherente con los requerimientos de negocio y ajustar si es necesario.
      Validar el diagrama con los stakeholders relevantes para asegurarse de que sea adecuado.
