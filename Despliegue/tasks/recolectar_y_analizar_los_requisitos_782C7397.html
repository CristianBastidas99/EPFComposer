Task: Recolectar y analizar los requisitos
Recolectar información de los interesados, identificar y documentar los requisitos del sistema, y realizar un análisis inicial de su viabilidad y relevancia.
Purpose
Recopilar y analizar los requisitos del sistema de manera sistemática y completa.
Relationships
RolesPrimary Performer:
  • Analista de Negocio
Additional Performers:
InputsMandatory:
  • Requisitos funcionales y no funcionales del sistema
    		Optional:
    • None
    Outputs
    • Especificación de Requisitos
      Main Description
      En esta tarea, se llevará a cabo la recolección sistemática de información de los interesados para identificar y documentar los requisitos del sistema. Se realizarán entrevistas, se revisarán documentos y se recopilarán datos relevantes para asegurar una comprensión completa de las necesidades del sistema. A continuación, se realizará un análisis inicial de los requisitos, evaluando su viabilidad y relevancia para el éxito del proyecto. El objetivo es establecer una base sólida de requisitos claros y completos que orienten el diseño y desarrollo del sistema.
      Steps
      Realizar entrevistas y reuniones con los stakeholders para comprender sus necesidades y expectativas.
      Identificar los requisitos funcionales y no funcionales del sistema.
      Documentar los requisitos utilizando técnicas como diagramas de casos de uso, listas de requisitos, etc.
      Validar los requisitos con los stakeholders para asegurar su comprensión y acuerdo.
