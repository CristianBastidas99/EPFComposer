Task: Definir y especificar las relaciones de dominio
Definir y especificar las relaciones de dominio en el código del back-end del proyecto para su implementación técnica.
Purpose
Establecer y detallar las interacciones entre los diferentes dominios o áreas del proyecto en el código del back-end.
Relationships
RolesPrimary Performer:
  • Desarrollador Backend
Additional Performers:
  • Analista de negocio
  • Analista de requrimientos
InputsMandatory:
  • Relaciones de dominio
    		Optional:
    • None
    Outputs
    • Relaciones de dominio
      Main Description
      En esta tarea, se traducirán las relaciones de dominio definidas anteriormente en el código del back-end del proyecto. Esto implica establecer la estructura y las interacciones entre los diferentes componentes del sistema en el código fuente, utilizando el lenguaje de programación adecuado. Al definir y especificar las relaciones de dominio en el código del back-end, se proporciona una base técnica sólida para la implementación del proyecto y se asegura la coherencia entre la visión del proyecto y su implementación práctica.
      Steps
      Revisar los requerimientos funcionales y no funcionales del proyecto para identificar las entidades y relaciones de dominio.
      Diseñar un modelo de entidad-relación (ER) que represente las entidades y relaciones de dominio identificadas en el paso anterior.
      Transformar el modelo ER en un modelo de objetos que se utilizará en el código del back-end.
      Especificar las clases y relaciones de las entidades en el lenguaje de programación del proyecto.
      Implementar las clases y relaciones de las entidades en el código del back-end del proyecto.
      Verificar que las relaciones de dominio se hayan implementado correctamente realizando pruebas de integración con el front-end y la base de datos.
